Linear Algebra for Data Science

B The iGraph and GGally Packages

Here is some additional information on the use of the iGraph and GGally packages for creating and performing computations on graphs.

We can create a graph in several ways. Perhaps the simplest is by giving it a vector of vertex numbers. By default, igraph will pair up the numbers to represent an edge. Another default assumption is that the first number in each pair is the starting vertex of a directed edge.

library(igraph)
edges <- c(1,2, 2,3, 4,3, 2,4)
g1 <- graph(edges)
g1
## IGRAPH d018c35 D--- 4 4 -- 
## + edges from d018c35:
## [1] 1->2 2->3 4->3 2->4

The plot function will usually create a decent plot for an uncomplicated graph, though the default vertex size is somewhat small (Fig. B.1). There can be some randomness in the placement of the vertices, so you might wish to set the seed before plotting the graph.

set.seed(1729)
plot(g1, vertex.size = 30)

Figure B.1: A graph

To create an undirected graph, just have directed = FALSE as a graph setting.

g1u <- graph(edges, directed = FALSE)
g1u
## IGRAPH d0304c2 U--- 4 4 -- 
## + edges from d0304c2:
## [1] 1--2 2--3 3--4 2--4

You can add an edge or edges using the add.edges function (Fig. B.2).

set.seed(1729)
g2u <- add.edges(g1u,c(1,1, 4,2))
plot(g2u, vertex.size = 30)

Figure B.2: The original graph with more edges, including a loop

You can find an adjacency matrix from a graph or create a graph from an adjacency matrix. By default, the output or input is a sparse matrix, with only the nonzero terms stored.

as_adjacency_matrix(g1)
## 4 x 4 sparse Matrix of class "dgCMatrix"
##             
## [1,] . 1 . .
## [2,] . . 1 1
## [3,] . . . .
## [4,] . . 1 .
as_adjacency_matrix(g1, sparse = FALSE)
##      [,1] [,2] [,3] [,4]
## [1,]    0    1    0    0
## [2,]    0    0    1    1
## [3,]    0    0    0    0
## [4,]    0    0    1    0
as_adjacency_matrix(g1u, sparse = FALSE)
##      [,1] [,2] [,3] [,4]
## [1,]    0    1    0    0
## [2,]    1    0    1    1
## [3,]    0    1    0    1
## [4,]    0    1    1    0
set.seed(1234)
# We create a random 5x5 matrix with zeros and ones.
M <- matrix(sample(0:1, 25, replace = TRUE), nrow = 5)
M
##      [,1] [,2] [,3] [,4] [,5]
## [1,]    1    1    1    1    1
## [2,]    1    0    1    1    1
## [3,]    1    0    1    0    1
## [4,]    1    0    0    1    1
## [5,]    0    1    1    1    1
gM <- graph_from_adjacency_matrix(M)
gM
## IGRAPH d04db13 D--- 5 19 -- 
## + edges from d04db13:
##  [1] 1->1 1->2 1->3 1->4 1->5 2->1 2->3 2->4 2->5 3->1 3->3 3->5 4->1 4->4 4->5
## [16] 5->2 5->3 5->4 5->5
plot(gM, vertex.size = 30, vertex.color = "lightblue")

Figure B.3: A graph from a random matrix

You can create a graph using named vertices instead of numbers (Fig. B.4)

edgesn <- c("Al", "Bob", "Al", "Cam", "Bob", "Cam")
gn <- graph(edgesn)
gn
## IGRAPH d05deed DN-- 3 3 -- 
## + attr: name (v/c)
## + edges from d05deed (vertex names):
## [1] Al ->Bob Al ->Cam Bob->Cam
plot(gn, vertex.size = 0)

Figure B.4: A graph with named labels

If you’re entering the graph by hand, it’s probably easier to do the naming later. You can either feed the plot function a vector of names, or you can change them using the V function. Here, we’ll change the names in the first graph.

V(g1)$name <- c("Bears", "Lions", "Packers", "Vikings")
g1
## IGRAPH d018c35 DN-- 4 4 -- 
## + attr: name (v/c)
## + edges from d018c35 (vertex names):
## [1] Bears  ->Lions   Lions  ->Packers Vikings->Packers Lions  ->Vikings

There are some ways to set the layout. For example, the following code will arrange the vertices of our random graph in a circle (Fig. B.5). Since there are five vertices, it looks like a pentagon.

coords <- layout_in_circle(gM)
plot(gM, vertex.size = 30, vertex.color = "lightblue",
     layout = coords)

Figure B.5: A circular layout

GGally

The GGally package is an extension to ggplot2 that includes many features. One of them is a ggnet2 function that will create nice graphs, as in Figure B.6. This graph uses the randomly generated adjacency matrix from above.

library(GGally)
set.seed(22)
ggnet2(M, arrow.size = 12,
       color = "orangered",
       arrow.gap = 0.025, 
       label = LETTERS[1:5])

Figure B.6: A graph using ggnet2

We can use ggnet2 with an igraph graph, though it requires the intergraph package. Figure B.7 shows the result.

library(intergraph)
set.seed(222)
ggnet2(gn, arrow.size = 12,
       color = "lightpink",
       arrow.gap = 0.045,
       size=15,
       label = V(gn)$name)

Figure B.7: Another ggnet2 graph
